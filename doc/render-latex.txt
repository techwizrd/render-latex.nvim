INTRODUCTION                                      *render-latex-introduction*

render-latex.nvim renders Markdown display math blocks as transparent PNG
images, with a fast inline text fallback for `$...$` and `\(...\)` spans.

This is an early release. Defaults are designed to be low-config and stable,
but option names may still change before v1.0.0.

Display math is rendered for:
>
  $$
  x^2 + y^2 = z^2
  $$
<
and:
>
  \[
  \int_0^\infty e^{-x^2}\,dx
  \]
<

Inline math is intentionally not image-rendered. It uses highlights and conceal
for smooth editing and predictable cursor behavior.

Images are hidden while completion popups or focusable floating windows are
open. Plain command-line typing does not hide images by default; opt in with
`render.hide_on_cmdline = true`. When command-line hiding is disabled, cmdline
UI floats are ignored but completion popups still hide images until
command-line mode exits. Non-focusable utility floats are ignored.

===============================================================================
REQUIREMENTS                                      *render-latex-requirements*

- Neovim 0.10+ with `vim.system`, `vim.uv`, `vim.fs`, extmarks, and Treesitter.
- An image backend: `vim.ui.img` support or a Kitty graphics-capable terminal.
- Rust toolchain only for development or source builds. Normal installs use
  prebuilt workers published with releases for Linux x64, macOS x64, macOS
  arm64, and Windows x64.
- For tmux image rendering: `set -g allow-passthrough on`.

The plugin installs the matching prebuilt worker in the background on first
setup. To install it manually:
>
  :RenderLatex install
<

To build from source instead:
>
  :RenderLatex build
<
or from a shell:
>
  make build-worker
<

SETUP                                                    *render-latex-setup*

Minimal lazy.nvim-style setup:
>
  {
    "techwizrd/render-latex.nvim",
    ft = "markdown",
    opts = {},
  }
<

Minimal vim.pack setup, on Neovim versions that include it:
>
  vim.pack.add({
    "https://github.com/techwizrd/render-latex.nvim",
  })
<

Common options:
>
  require("render_latex").setup({
    render = {
      preset = "match_text", -- "compact" or "presentation"
      inline = "conceal",    -- "content", "highlight", or false
      inline_symbols = true,
      live_preview = true,
      hide_on_cmdline = false,
      background = "transparent",
    },
    image = {
      backend = "auto",      -- "nvim" or "kitty"
    },
    install = {
      auto = true,
      repository = "techwizrd/render-latex.nvim",
      version = "latest",
    },
    tmux = {
      install_cleanup_hooks = false,
    },
  })
<

If you pin plugin versions, pin the worker release too:
>
  require("render_latex").setup({
    install = { version = "v0.1.0" },
  })
<

If you want rolling worker binaries from `main`, use the unreleased prerelease:
>
  require("render_latex").setup({
    install = { version = "unreleased" },
  })
<

For best inline fallback rendering:
>
  vim.opt.conceallevel = 2
  vim.opt.concealcursor = "nc"
<

OPTIONS                                             *render-latex-options*

Call `require("render_latex").setup({ ... })` with any of these keys. You do
not need to set anything for the default experience.

Top-level options:

                                                        *render-latex-enabled*
`enabled`                Enable rendering by default. Default: `true`.

                                                       *render-latex-debounce*
`debounce`               Delay in milliseconds before queued render work runs.
                         Default: `75`.

                                                     *render-latex-file_types*
`file_types`             Filetypes where the plugin attaches. Default:
                         `{ "markdown" }`.

                                                   *render-latex-render_modes*
`render_modes`           Modes where rendered images are shown. Default:
                         `{ "n", "i" }`.

                                                 *render-latex-prefetch_lines*
`prefetch_lines`         Number of lines around the viewport to render ahead.
                         The prefetch window follows scroll direction. Default:
                         `40`.

                                                       *render-latex-conceal*
`conceal`                Hide inline math delimiters when inline fallback is
                         active. Default: `true`.

                                                *render-latex-max_file_lines*
`max_file_lines`         Very large files skip display image rendering and keep
                         lightweight inline fallback behavior. Default: `5000`.

Worker options:

                                                     *render-latex-worker.bin*
`worker.bin`             Path to a worker binary you built or downloaded
                         yourself. Default: unset.

                                                    *render-latex-worker.args*
`worker.args`            Extra arguments passed to the worker. Default: `{}`.

Install options:

                                                   *render-latex-install.auto*
`install.auto`           Download the matching prebuilt worker in the background
                         when no worker is found. Default: `true`.

                                             *render-latex-install.repository*
`install.repository`     GitHub repository used for release downloads. Default:
                         `"techwizrd/render-latex.nvim"`.

                                                *render-latex-install.version*
`install.version`        Release version used for worker downloads. Use a tag
                         like `"v0.1.0"` when pinning the plugin, or
                         `"unreleased"` for rolling prerelease assets from
                         `main`. Default: `"latest"`.

Render options:

                                                *render-latex-render.preset*
`render.preset`          Sizing preset: `"match_text"`, `"compact"`, or
                         `"presentation"`. Default: `"match_text"`.

                                            *render-latex-render.foreground*
`render.foreground`      Explicit equation foreground color. When unset, the
                         current text color is used. Default: unset.

                                      *render-latex-render.match_text_color*
`render.match_text_color`
                         Use the current text color for rendered equations when
                         `render.foreground` is unset. Default: `true`.

                                           *render-latex-render.background*
`render.background`      Equation background color. Default: `"transparent"`.

                                      *render-latex-render.match_text_size*
`render.match_text_size`
                         Match rendered equation size to buffer text when
                         possible. Default: `true`.

                                             *render-latex-render.font_size*
`render.font_size`       Base font size sent to the worker. Default: `34`.

                                            *render-latex-render.text_scale*
`render.text_scale`      Scale applied when matching rendered output to text.
                         Default: `1.0`.

                                         *render-latex-render.exit_delay_ms*
`render.exit_delay_ms`   Short delay before rerendering after leaving an edited
                         equation. Default: `150`.

                                           *render-latex-render.placeholder*
`render.placeholder`     Show temporary virtual text while an edited equation is
                         rerendering. Default: `true`.

                                      *render-latex-render.equation_labels*
`render.equation_labels` Add equation labels. Values: `"right"`, `"sign"`,
                         `"both"`, or `false`. Default: `"right"`.

                               *render-latex-render.equation_label_format*
`render.equation_label_format`
                         Label text format. Default: `"Eq. %d"`.

                                               *render-latex-render.padding*
`render.padding`         Padding around rendered equations. Default: `10`.

                                                 *render-latex-render.scale*
`render.scale`           Render scale sent to the worker. Default: `1.5`.

                                                *render-latex-render.inline*
`render.inline`          Inline math fallback mode: `"conceal"`, `"content"`,
                         `"highlight"`, or `false`. Default: `"conceal"`.

                                         *render-latex-render.inline_symbols*
`render.inline_symbols`  Replace common inline LaTeX commands with readable
                         symbols in conceal mode. Default: `true`.

                                          *render-latex-render.live_preview*
`render.live_preview`   While editing a focused display equation, keep the raw
                         source visible and show a live rendered preview below
                         it. Default: `true`.

                                        *render-latex-render.hide_on_cmdline*
`render.hide_on_cmdline` Hide images while command-line mode is active. Default:
                         `false`.

Image options:

                                                *render-latex-image.backend*
`image.backend`          Image backend: `"auto"`, `"nvim"`, or `"kitty"`.
                         Default: `"auto"`.

                                                 *render-latex-image.zindex*
`image.zindex`           Z-index for image placement. Default: `60`.

                                          *render-latex-image.cell_width_px*
`image.cell_width_px`    Fallback terminal cell width in pixels for Kitty
                         placement. Default: `10`.

                                         *render-latex-image.cell_height_px*
`image.cell_height_px`   Fallback terminal cell height in pixels for Kitty
                         placement. Default: `20`.

Tmux options:

                                  *render-latex-tmux.install_cleanup_hooks*
`tmux.install_cleanup_hooks`
                         Install session-local cleanup hooks automatically.
                         This is opt-in because it mutates tmux state. Default:
                         `false`.

Integration options:

                                *render-latex-integrations.jupynvim.enabled*
`integrations.jupynvim.enabled`
                         Enable the experimental jupynvim source integration.
                         When enabled, notebook Markdown cells render display
                         math as images and code cells are ignored. Default:
                         `true`.

COMMANDS                                             *render-latex-commands*

All commands are subcommands of `:RenderLatex`.

                                                        *:RenderLatex-enable*
`:RenderLatex enable`       Enable rendering in supported buffers.

                                                       *:RenderLatex-disable*
`:RenderLatex disable`      Disable rendering in the current buffer.

                                                        *:RenderLatex-toggle*
`:RenderLatex toggle`       Toggle rendering.

                                                       *:RenderLatex-refresh*
`:RenderLatex refresh`      Queue a fresh render for the current buffer.

                                                        *:RenderLatex-install*
`:RenderLatex install`      Download the prebuilt worker for this platform.

                                                          *:RenderLatex-build*
`:RenderLatex build`        Build the Rust worker from source.

                                                        *:RenderLatex-status*
`:RenderLatex status`       Show raw status data with backend, worker, and
                            integration diagnostics.

                                                        *:RenderLatex-doctor*
`:RenderLatex doctor`       Open a readable diagnostics scratch buffer.

                                                    *:RenderLatex-tmux_check*
`:RenderLatex tmux_check`   Open tmux diagnostics.

                                                    *:RenderLatex-tmux_setup*
`:RenderLatex tmux_setup`   Show optional tmux setup lines.

                                                *:RenderLatex-equation_debug*
`:RenderLatex equation_debug`    Inspect the equation under the cursor.

                                             *:RenderLatex-equation_rerender*
`:RenderLatex equation_rerender` Queue rerender for the equation under cursor.

                                              *:RenderLatex-equation_source*
`:RenderLatex equation_source`   Open source for the equation under cursor.

                                              *:RenderLatex-equation_toggle*
`:RenderLatex equation_toggle`   Toggle raw/rendered state for the equation.

COMPATIBILITY                                   *render-latex-compatibility*

OBSIDIAN.NVIM                                      *render-latex-obsidian*

No special configuration is required. render-latex.nvim ignores YAML
frontmatter, fenced code blocks, inline code spans, Obsidian comments, and
indented code blocks. Display math inside Markdown blockquotes and Obsidian
callouts is supported.

RENDER-MARKDOWN.NVIM                        *render-latex-render-markdown*

Let render-markdown.nvim handle Markdown structure and let render-latex.nvim
own LaTeX rendering:
>
  require("render-markdown").setup({
    latex = { enabled = false },
  })
<

LazyVim users can apply the same override in a plugin spec:
>
  return {
    {
      "MeanderingProgrammer/render-markdown.nvim",
      opts = {
        latex = { enabled = false },
      },
    },
  }
<

render-latex.nvim does not mutate other plugins' configuration. Compatibility
diagnostics only run in manual commands and health checks, not in the render
loop. Diagnostics are scoped to the current buffer and report whether
render-markdown.nvim is loaded, enabled, and inspectable.

JUPYNVIM EXPERIMENTAL                         *render-latex-jupynvim*

render-latex.nvim has an experimental source integration for jupynvim. When a
jupynvim notebook buffer is detected, display math inside Markdown cells is
rendered as images. Code cells are ignored, and inline math inside notebook
Markdown cells is left to jupynvim for now.

Disable this integration without disabling regular Markdown rendering:
>
  require("render_latex").setup({
    integrations = {
      jupynvim = { enabled = false },
    },
  })
<

If jupynvim cell metadata and buffer separators disagree, render-latex.nvim
skips notebook rendering for safety and reports the mismatch in `:RenderLatex
doctor` and `:checkhealth render_latex`. Cell type changes are picked up through
the source revision path on the next render queue; stable upstream jupynvim cell
range APIs and metadata-change events would allow more direct integration.

Other plugins can expose Markdown-like regions through the experimental source
API:
>
  require("render_latex").register_source({
    name = "my-plugin",
    attach = function(bufnr)
      return true -- return true only for buffers this source owns
    end,
    display_ranges = function(bufnr)
      return {
        { start_row = 0, end_row = 10 },
      }
    end,
    revision = function(bufnr)
      return "optional-source-state-version"
    end,
    inline = false,
  })
<

Use revision for non-incremental sources whose external state can change without
buffer text changes.

This API is intentionally experimental and may change before v1.0.0.

TMUX                                                      *render-latex-tmux*

For image rendering in tmux:
>
  set -g allow-passthrough on
  set -g focus-events on
  set -g default-terminal "tmux-256color"
<

Session-local cleanup hooks are opt-in:
>
  require("render_latex").setup({
    tmux = { install_cleanup_hooks = true },
  })
<

Use `:RenderLatex tmux_check` and `:RenderLatex tmux_setup` for diagnostics and
setup details.

===============================================================================
TROUBLESHOOTING                              *render-latex-troubleshooting*

Run:
>
  :RenderLatex doctor
  :checkhealth render_latex
<

Common issues:

- Worker missing: run `:RenderLatex install`, `:RenderLatex build`, or
  configure `worker.bin`.
- Worker auto-install fails: check proxy/firewall/offline restrictions. You can
  disable background install with `install.auto = false` and retry manually with
  `:RenderLatex install`.
- Linux ARM64 does not ship a prebuilt worker in the first release. Use
  `:RenderLatex build` or configure `worker.bin` there.
- No display images: check the image backend and terminal support in doctor.
- Inline math is not concealed: set `conceallevel=2`.
- render-markdown.nvim overlaps math: set `latex = { enabled = false }` there.
- tmux images do not appear: enable `allow-passthrough` and run tmux diagnostics.
